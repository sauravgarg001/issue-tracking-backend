Events to listen

verifyUser event
This event ("verifyUser") has to be listened on the user's end to verify user authentication.
user will only be set as online user after verification of authentication token.
auth-error@<userId> event
This event ("auth-error@<userId>") has to be listened on the user's end to identify authentication error or breach in security.
The output will error message.
auth-error event
This event ("auth-error") has to be listened on the user's end to identify authentication error or breach in security.
The output will error message.
disconnect event
This event ("disconnect") has to be listened, when the user is disconnected.
issues@<userId> event
This event ("issues@<userId>") has to be listened on the user's end to get notified that issues need to be updated.
No data passed
issues-assigned@<userId> event
This event ("issues-assigned@<userId>") has to be listened on the user's end to get notified that assigned issues need to be updated.
No data passed
issues-reported@<userId> event
This event ("issues-reported@<userId>") has to be listened on the user's end to get notified that reported issues need to be updated.
No data passed
comments@<userId> event
This event ("comments@<userId>") has to be listened on the user's end to get notified that comments on issue needs to be updated.
issueId in data is passed
watchers@<userId> event
This event ("watchers@<userId>") has to be listened on the user's end to get notified that wathcers of issue need to be updated.
issueId in data is passed
watchers-count@<userId> event
This event ("watchers-count@<userId>") has to be listened on the user's end to get notified that issue watchers count need to be updated.
issueId in data is passed
notifications@<userId> event
This event ("notifications@<userId>") has to be listened on the user's end to get notified that notifications need to be updated.
No data passed
